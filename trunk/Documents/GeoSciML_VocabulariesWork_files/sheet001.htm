This spreadsheet summarizes all properties in the GeoSciML v2 schema (as of 19 Sept 2007) that may be specified by a controlled concept. Columns are as follows: 
AttributeName Name of the attribute in the model. This will appear as an element tag contained in a sequence element identified by the className column in XML instance documents. This is an attribute in the UML diagram.
ClassName Name of element that contains attribute element named in AttributeName column in XML instance documents. This is a class in the UML diagram.
DataType Datatype associated with attribute in UML diagram. This will be the name of an element contained in the AttributeName element in XML instance documents.
priority Ranking of priority for developing vocabulary. Rank 1 is for vocabularies for required elements in documents required by Use cases identified for testbed 3. Rank 2 is for optional elements deemed (by CDTG members...) to be more likely to be populated in instance documents for testbed 3. Rank 3 is for optional associated elements or optional elements deemed less likely to be used in Testbed 3. Rank 4 is for elements that were deemed outside scope for Testbed 3.
VocabName name of vocabulary, used to name excel spreadsheets for term compilation and definition development
Model Stereotype Stereotype associated with element in UML model.
Shepherd Person responsible for compiling vocabulary
Status Current development status of vocabulary
Notes notes on development process
Status and notes will be updated periodically by the task group leader.
