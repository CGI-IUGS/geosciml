XmlServiceTester

Name

XmlServiceTester - schema-validate responses from XML services

Synopsis

XmlServiceTester.sh test-suite.xml

XmlServiceTester.bat test-suite.xml

Description

XmlServiceTester is a command-line tool that obtains XML responses from web services or local files and uses the Xerces-2 SAX parser to schema-validate these responses. Rudimentary content validation (element counting) is also supported.

XmlServiceTester is configured with a test-suite XML document that conforms to the schema:

  • http://schemas.auscope.org/servicetester/test-suite-1.0.xsd
All file paths must be URIs (i.e. forward slashes). Relative URIs are relative to the containing document.

Validation

Any warnings are considered errors. Schema are validated as well as the XML documents.

Return code

XmlServiceTester exits with return code 0 on success, and with return code 1 if any error occurs.

Requirements

Java 5 or later.

Example

See the example test-suite file:
  • test-suite.example.xml

Issues

  • HTTP GET not supported
  • Validation of HTTP POST request not performed
  • No-namespace schema location support incomplete
  • Request bodies not schema-validated
